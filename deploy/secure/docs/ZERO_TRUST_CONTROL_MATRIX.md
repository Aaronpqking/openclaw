# Zero-Trust Control Matrix

| Layer              | MVP control                              | V1.1 control                                                                                                             | Enforcement point                                    | Validation method                                                                  | Failure mode                                    | Priority |
| ------------------ | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------- | ---------------------------------------------------------------------------------- | ----------------------------------------------- | -------- |
| Identity           | OpenClaw native token/password auth      | Identity-aware proxy auth plus `gateway.auth.mode="trusted-proxy"` and `allowUsers`                                      | host nginx auth gate and OpenClaw trusted-proxy auth | authenticate through proxy, verify allowed user succeeds and disallowed user fails | unauthorized browser session reaches Control UI | critical |
| Device             | Tailscale-only remote admin path         | Tailscale device identity plus proxy identity                                                                            | Tailscale tailnet ACL/device posture and host access | verify access only from approved tailnet device                                    | stolen or unmanaged device reaches proxy        | high     |
| Ingress            | Tailscale Serve to host loopback nginx   | same, with host nginx only                                                                                               | Tailscale Serve and host loopback bind               | `tailscale serve status`, remote browser test                                      | direct internet/LAN ingress                     | critical |
| Reverse proxy      | nginx container on `127.0.0.1:8080`      | host nginx on `127.0.0.1:8080` with `auth_request` and header overwrite                                                  | nginx config                                         | inspect config, `curl -I`, WebSocket connect                                       | spoofed forwarded headers or auth bypass        | critical |
| Browser/CSP        | CSP Report-Only                          | CSP Report-Only until promotion criteria pass                                                                            | nginx response headers                               | browser devtools and CSP report review                                             | UI breakage or silent CSP bypass                | medium   |
| Container runtime  | hardened Docker container                | hardened Docker container, optional gVisor override                                                                      | Docker and optional `runsc`                          | `docker compose config`, `docker inspect`, `dmesg` in gVisor container             | broader syscall/kernel attack surface           | high     |
| Filesystem         | host-mounted `~/.openclaw` and workspace | same, with dedicated host and tight perms                                                                                | host filesystem perms and Docker mounts              | `openclaw security audit --deep`, `ls -ld ~/.openclaw*`                            | state tampering or secret disclosure            | critical |
| Secrets            | env vars or config-managed secrets       | same, plus host-only reverse-proxy secrets                                                                               | shell env, config, proxy secret store                | confirm secrets absent from repo and logs                                          | leaked gateway/proxy secrets                    | critical |
| Extensions/plugins | operator discretion                      | explicit production deny-by-default stance, `plugins.allow` allowlist, no marketplace/community installs unless approved | OpenClaw config and operator process                 | `openclaw plugins list`, inspect `plugins.allow`, review config                    | in-process untrusted code execution             | critical |
| Outbound network   | unrestricted host/container egress       | host-level egress allowlist scaffold with maintenance windows                                                            | nftables or equivalent host firewall                 | `nft list ruleset`, provider reachability checks                                   | arbitrary exfiltration or broad outbound abuse  | high     |
| Audit/logging      | container logs and manual checks         | same, plus auth proxy logs and security audit cadence                                                                    | Docker logs, nginx logs, `openclaw security audit`   | check `/healthz`, `/readyz`, audit output, auth logs                               | drift goes unnoticed                            | high     |
| Admin workflow     | manual compose + Tailscale Serve         | documented change control for proxy/auth/firewall/gVisor changes                                                         | operator runbook                                     | follow validation checklist before and after changes                               | emergency edits widen trust silently            | high     |
